This description is used as AI context throughout your entire project — the AI architect will read it before every conversation to understand your goals, constraints, and design intent. The richer and more specific it is, the better the AI guidance you will receive.

Example: A global API service handling identity verification for a consumer web app. Expected 10k daily users worldwide, storing PII data with GDPR compliance, requiring high availability and low latency.

💡 Descriptions over 500 characters give the AI enough context to provide meaningful architecture guidance.

A good description includes:

  1. What type of application are you building and what problem does it solve?
  2. Who will use it, from where, and how many users do you expect?
  3. What main components will it include (web app, API, database, etc.)?
  4. What type of data will it handle — including any sensitive or regulated data?
  5. Any specific requirements: high availability, compliance (GDPR, HIPAA), latency, cost constraints?
